# Hockey IRT Dashboard — Requirements

## Overview
A Streamlit dashboard deployed to Streamlit Community Cloud that displays
goalie and shooter skill estimates from the Dynamic IRT model.  Data is
pre-aggregated into small CSVs and committed to the repository; the dashboard
queries those files directly.  Updates are manual: re-run the model locally
and push new CSVs.

---

## Data Architecture

### Pre-aggregated export files (committed to `dashboard/data/`)

| File | Rows | Description |
|---|---|---|
| `goalie_summary.csv` | ~120 rows | One row per goalie. Season-level stats. |
| `goalie_weekly.csv` | ~2,500 rows | One row per (goalie, week). Skill trajectory. |
| `shooter_summary.csv` | ~1,500 rows | One row per shooter. Season-level stats. |
| `shooter_weekly.csv` | ~55,000 rows | One row per (shooter, week). Skill trajectory. |

These are generated by a new script `dashboard/build_data.py` (see §Build).

## Build Script — `dashboard/build_data.py`

Generates all four CSVs from model state + name cache + NHL API team lookups.

```
python dashboard/build_data.py [--state model_output/state_latest.pkl]
                               [--cache model_output/player_names_cache.json]
                               [--out   dashboard/data/]
                               [--season-idx 1]
```

Steps:
1. Load `ModelState` from `--state`
2. Call `compute_gsax(state, season_idx)` → `goalie_summary`
3. Call `summarize_skills(state, season_idx)` → `goalie_weekly`, `shooter_weekly`
4. Aggregate `shooter_weekly` → `shooter_summary`
5. Fetch team + position for all player IDs via NHL API (cached in `--cache`)
6. Merge names, teams, positions into all four tables
7. Write CSVs to `--out`

## Dashboard Structure

Single-file app: `dashboard/app.py` with a sidebar page selector.

### Pages

#### 1. Goalie Leaderboard
- Full-season table sorted by `gsax` descending by default
- Columns displayed: Player, Team, Shots Faced, GSAx, GSAx Raw, Shooter Adj,
  Save %, xG Save %
- Filter controls: Team multiselect, min shots faced slider (default 200)
- Column header click → resort
- Row click (or selectbox) → jumps to Goalie Trajectory page for that player

#### 2. Goalie Skill Trajectory
- Player selector (searchable dropdown)
- Line chart: `phi` over weeks for selected goalie(s) (up to 5 at once)
- Secondary line: `mu_phi` as a dashed horizontal reference
- **Confidence shading**: band width inversely proportional to `shots_that_week`
  (wider band = fewer shots faced = less certain estimate)
- Tooltip: week, phi, mu_phi, shots_that_week
- Comparison mode: overlay multiple goalies on one chart

#### 3. Shooter Leaderboard
- Table sorted by `mu_theta` descending by default; also sortable by `fsax`
- Columns: Player, Team, Position, Season Skill (mu_theta), FSAx, FSAx Raw,
  Goalie Adj, Latest Week, Season High, Season Low
- Filters: Team multiselect, Position multiselect (C/LW/RW/D/all),
  min shots taken slider (default 20)

#### 4. Shooter Skill Trajectory
- Same structure as Goalie Trajectory but for `theta`
- Player selector, line chart, comparison mode
- **Confidence shading**: band width inversely proportional to `shots_that_week`

#### 5. Head-to-Head
- Two selectors: one goalie, one shooter
- Shows both skill trajectories on one chart (dual-axis or normalised)
- Summary card: goalie's φ vs shooter's θ at latest week — who has the edge?

## Deployment

- **Platform**: Streamlit Community Cloud (free tier)
- **Repository**: Public GitHub repo (or existing private repo with Streamlit
  access granted)
- **Entry point**: `dashboard/app.py`
- **Dependencies**: `dashboard/requirements.txt`
  - `streamlit`
  - `pandas`
  - `altair` (charts)
  - `requests` (for any live API calls, optional)
- **Secrets**: None required (all data is static CSVs)
- **Data path**: relative paths from `dashboard/` root, i.e.
  `data/goalie_summary.csv`

### Deployment steps
1. Push `dashboard/` directory (including `data/` CSVs) to GitHub
2. Connect repo to Streamlit Community Cloud
3. Set entry point to `dashboard/app.py`
4. Done — no environment variables needed

### Updating data
1. Re-run model: `python run_model.py`
2. Rebuild dashboard data: `python dashboard/build_data.py`
3. `git add dashboard/data/ && git commit -m "update data" && git push`
4. Streamlit Cloud auto-redeploys on push

## Future Work

- **Multi-season trajectories**: overlay 2024-25 and 2025-26 skill curves on
  the same chart for historical context. Deferred — single-season view only
  for now.
- **GitHub Actions automation**: scheduled workflow that runs the full
  ingest → xG → model → build_data pipeline and commits updated CSVs,
  removing the need for manual pushes.
- **Rolling shot-count shading**: currently confidence bands use the raw
  weekly shot count. A smoothed 4-week rolling average could look cleaner.
- **Team history**: players who changed teams mid-season are shown with their
  current team only. Future: show team at time of each shot.
